Projects

This is what I've done.

Here you can read about some projects that I have developed - whether it be on my own free time, at hackathons, for work, or for school. However, this is not a complete list. Feel free to check out my GitHub to see more projects that are not listed below.

  • Speaking Sensibly

    Speaking Sensibly is a project that I had the honor of being a part of during the Major League Hacking Prime Spring 2016 Hackathon. The project consisted of a mobile app client as well as a web application portal. Speaking Sensibly revolved around the Tanvas touch technology that allowed users to feel texture on smooth surface using haptic feedback. The goal of Speaking Sensibly is to help patients diagnosed with Autism Spectrum Disorder familiarize with objects that they may be sensitive to.

    This project was awarded 2nd place for the Best Touch-controlled UI, presented by Tanvas.

    The web application portion of the project was written in TypeScript, HTML, CSS. The frameworks that were used for this projects were Express and Angular 2.0.

    A much more detailed description of this project is available here.

  • coverbot

    coverbot is a bot I developed for my Slack team at Rutgers Office of Information Technology. The purpose of the bot is to manage the communication between employees regarding shift coverages. The bot is able to post announcements and direct message users based on commands sent to the bot. This was a very fun project for me because I was able to scaffold the entire project from scratch - the only resource that I utilized was the official Slack Client API documentation. I also created parsing and validation functions, including unit testing functions for the bot.

    The entire bot was written using JavaScript. The only frameworks that I used for the bot are Mocha and Chai for unit testing.

    Source code and documentation can be found on my GitHub here.

  • Photo Album

    Photo Album is a project that I developed for a class named Software Methodologies. The project is a fully functional Photo Album that features a admin panel, user log in, as well as persistent data. The administrator is has the ability to create or delete users. Users have the ability to create albums and upload photos with optional tags. The user can also search for photos by tag and create a new photo album using the search results. When the user logs out of their account, all the data is saved and persists even if the program is closed.

    The class Software Methodologies places an emphasis on proper software development practices and techniques. As a result, this project was written keeping the best practices in mind. The photo album was written completely in Java 8 and JavaFX. Some practices that I kept in mind while writing the photo album included object oriented programming, MVC design pattern, singletons, and modular code.

    The source code for this project can be found here.

  • SOCRATES

    SOCRATES stands for SOcial and CRowdsourced AcTivities Extraction System. The intention of this platform is to be a one stop shop for researchers interested in studying social media. Users are able to query data from several social media platforms such as Twitter, Facebook, Youtube, New York Times, and Reddit. In addition to aggregating the data, SOCRATES also allows the user to analyze the data using statistical and textual functions as well as visually explore the data using graphs.

    The languages that I came across in this project was JavaScript, HTML, CSS, Python, and php. MongoDB was the database used and d3.js produced the graphs.

    You can find out more information about this project here.

  • Multiprocess Bank Server/Client

    An awesome project that I did for a class called Systems Programming! Normally, I wouldn't want to include coursework on here, but I had such a great time on this one and learned SO much from this one project that I just had to.

    This program started a bank server that contained persistent data for a bank. Clients are then able to connect to the server via the hostname and port number. Each successful connection spawns a new process that handles the client session. During a client session, the client is able to open, start, credit, debit, and retrieve the balance for a bank account. A bank account may only be in session by one client at a time, and all other clients will wait on that account until the first client finishes its session.

    The functionality of the bank accounts was simple. The exciting part about this project was socket programming, multithreaded programming, multiprocess programming, mutex locks, shared memory, and memory maps. Lastly, I was able to keep with good software design principles: each function/thread had one responsibility and they were able to change internally without effecting other parts of the code.

    Take a look at the source code here.
