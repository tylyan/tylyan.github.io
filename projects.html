Projects

This is what I've done.

Below are a few projects that I thought worthy enough to mention. Other projects not listed mostly include school assignments and small test programs.

  • SOCRATES

    SOCRATES stands for SOcial and CRowdsourced AcTivities Extraction System. The intention of this platform is to be a one stop shop for researchers interested in studying social media. Users are able to query data from several social media platforms such as Twitter, Facebook, Youtube, New York Times, and Reddit. In addition to aggregating the data, SOCRATES also allows the user to analyze the data using statistical and textual functions as well as visually explore the data using graphs.

    The languages that I came across in this project was JavaScript, HTML, CSS, Python, and php. MongoDB was the database used and d3.js produced the graphs.

    You can find out more information about this project here.

  • Multiprocess Bank Server/Client

    An awesome project that I did for a class called Systems Programming! Normally, I wouldn't want to include coursework on here, but I had such a great time on this one and learned SO much from this one project that I just had to.

    This program started a bank server that contained persistent data for a bank. Clients are then able to connect to the server via the hostname and port number. Each successful connection spawns a new process that handles the client session. During a client session, the client is able to open, start, credit, debit, and retrieve the balance for a bank account. A bank account may only be in session by one client at a time, and all other clients will wait on that account until the first client finishes its session.

    The functionality of the bank accounts was simple. The exciting part about this project was socket programming, multithreaded programming, multiprocess programming, mutex locks, shared memory, and memory maps. Lastly, I was able to keep with good software design principles: each function/thread had one responsibility and they were able to change internally without effecting other parts of the code.

    Take a look at the source code here.

  • Chess for Android

    A playable chess game for the Android platform! I wrote the game at first in Java to run in the terminal. Later on, I was able to port the game over to Android seamlessly because I followed the MVC design pattern. Working on this project definitely taught me more about object oriented programming, abstract classes, inheritence, Android development, and software design principles.
